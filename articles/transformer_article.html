01 — Overview

What is a Transformer, really?

In 2017, a team at Google published a paper with a provocative title: "Attention Is All You Need." The architecture they described — the Transformer — went on to power GPT, BERT, T5, Claude, Gemini, and virtually every frontier AI system you interact with today.

Before Transformers, sequence models were sequential. RNNs and LSTMs processed tokens one at a time, left to right, carrying a hidden state forward like a rolling memory. This meant long-range dependencies were hard to learn, training was slow, and parallelisation was nearly impossible.

"What if we just let every word attend directly to every other word — all at once?"

— The insight behind self-attention

The Transformer's answer was radical: process all tokens in parallel, and use attention to let each token decide which other tokens are relevant. No recurrence. No convolution. Just attention.

  • All tokens processed simultaneously — no sequential bottleneck
  • Attention replaces recurrence for capturing dependencies
  • Residual connections + LayerNorm enable very deep stacking
  • Positional encoding preserves word-order information

02 — Core Mechanism

Self-attention: every word talks to every other word

Self-attention is the engine of the Transformer. For each token, it asks: "Which other tokens in this sequence are relevant to understanding me?"

Every token generates three vectors by multiplying its embedding against three learned weight matrices:

  • Query (Q) — "What am I looking for?"
  • Key (K) — "What do I offer to others?"
  • Value (V) — "What information do I actually carry?"

The attention score between two tokens is computed by taking the dot product of one token's Query with another token's Key. A high dot product means "these two are relevant to each other." Scores are then scaled by √d_k to prevent them from getting too large, passed through a softmax (so they sum to 1), and used as weights to combine all the Values.

Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V

The result is a new vector for each token — one that now contains a weighted blend of context from the entire sequence. The word "bank" after reading "river" will look very different from "bank" after reading "loan." That context-sensitivity is entirely captured by attention.

03 — Multi-Head Attention

Multiple heads, multiple lenses on the same data

Running attention once gives you one perspective. But language has multiple types of relationships happening simultaneously — a word can be syntactically linked to one word, semantically related to another, and co-referential with a third.

Multi-head attention runs h independent attention operations in parallel, each with its own learned weight matrices Wᵢ_Q, Wᵢ_K, Wᵢ_V. Each head projects the input into a lower-dimensional space (d_model / h) — so the total compute stays constant — but learns to capture different kinds of patterns.

All head outputs are concatenated and passed through a final linear projection W_O to produce the multi-head output.

MultiHead(Q,K,V) = Concat(head₁, …, headₕ) · W_O

04 — The Encoder

The encoder: building a rich representation of the input

The encoder reads the entire input sequence and transforms it into a set of contextual representations — one vector per token, each aware of its full context. In the original paper, 6 identical encoder layers are stacked.

Each encoder layer has exactly two sub-layers:

Sub-layer 1 — Multi-head self-attention

Every token attends to every other token (bidirectional — no masking). After attention, the output is added back to the original input (residual connection) and then normalised (LayerNorm). This "Add & Norm" pattern is critical: the residual lets gradients flow cleanly through dozens of stacked layers without vanishing, while LayerNorm keeps activations well-scaled during training.

Sub-layer 2 — Feed-forward network (FFN)

Each token's vector passes independently through a two-layer MLP: Linear → ReLU → Linear. The hidden dimension is typically 4× the model dimension. This is where the model does "position-wise reasoning" — additional computation per token that goes beyond what attention can capture. Same weights are applied to every token position.

  • Attention sub-layer: tokens interact with each other
  • FFN sub-layer: each token processes independently
  • Add & Norm after every sub-layer — crucial for deep stacking
  • Positional encoding added at the very bottom before layer 1

05 — The Decoder

The decoder: generating output one token at a time

The decoder generates output autoregressively — one token per step, feeding each generated token back as input for the next. It has three sub-layers instead of two:

Sub-layer 1 — Masked self-attention

Same as encoder self-attention, but with a causal mask: each position can only attend to positions before it (including itself). This prevents the decoder from "cheating" by peeking at future tokens during training. At inference, there are no future tokens to peek at anyway.

Sub-layer 2 — Cross-attention (encoder-decoder attention)

This is the bridge between encoder and decoder. Queries come from the decoder's current state; Keys and Values come from the encoder's output. The decoder is essentially asking: "Given what I've generated so far, which part of the input should I focus on right now?"

This is where translation and summarisation happen — the decoder learns to selectively pull information from the encoder at each step of generation.

Sub-layer 3 — Feed-forward network

Identical to the encoder FFN. Add & Norm after each sub-layer, same as before.

"Cross-attention is where the two halves of the Transformer talk to each other. It's the moment of translation."

07 — Modern Variants

Three families of Transformers in the wild

The original paper used a full encoder-decoder architecture. But since 2017, the community has found that different tasks benefit from different parts of the architecture:

Encoder-only

Full bidirectional attention. Every token sees every other token. Deep language understanding.

BERT, RoBERTa

Best for: classification, NER, question answering, embeddings.

Decoder-only

Causal (left-to-right) attention only. Generates text autoregressively, token by token.

GPT, Claude, Llama

Best for: text generation, chat, code completion, reasoning.

Encoder-decoder

Full encoder + cross-attending decoder. Maps input sequences to output sequences.

T5, BART, original paper

Best for: translation, summarisation, structured generation.

The decoder-only architecture has dominated frontier AI since GPT-2. It's simpler, scales well, and — crucially — can be adapted to virtually any task through prompting, fine-tuning, or RLHF. Most models you interact with today (ChatGPT, Claude, Gemini) are decoder-only Transformers at heart.

What comes next?

The Transformer's one weakness is quadratic complexity: self-attention is O(n²·d) in sequence length. For long documents or high-resolution images, this becomes expensive. Research directions like Mamba (state-space models), linear attention, and FlashAttention are all attacking this bottleneck — but the core attention mechanism remains at the centre of nearly every approach.

"Understanding Transformers deeply is no longer optional for anyone working in AI. It's the foundation everything else is built on."
